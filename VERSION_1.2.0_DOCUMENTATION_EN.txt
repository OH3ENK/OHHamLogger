A "shitlog" is a poorly formatted ADI file that doesn't follow 
proper ADI 3.1.0 standards but is still readable by some logging 
software. Common issues:

âŒ All fields on one line (not separate lines)
âŒ Uses DATE instead of QSO_DATE
âŒ Uses WWFF_REF instead of SIG_INFO
âŒ Uses MY_WWFF_REF instead of MY_SIG_INFO
âŒ Missing proper field spacing
âŒ Incorrect field ordering

OHHamLogger v1.2.0 can now read these "shitlogs" by:
âœ“ Supporting both DATE and QSO_DATE
âœ“ Supporting both WWFF_REF and SIG_INFO
âœ“ Supporting both MY_WWFF_REF and MY_SIG_INFO
âœ“ Flexible field parsing (fields can be on one line)

This ensures compatibility with logs created by older or 
non-compliant logging software.

Q: WWFF reference appears in comment field after loading log
A: The log was created with old version. Edit the QSO and remove 
   WWFF from comment field manually.

Q: Cannot open old ADI log
A: Try using "shitlog" compatibility mode - v1.2.0 supports 
   alternative field names (DATE, WWFF_REF, MY_WWFF_REF).

Q: Time is wrong
A: All times are in GMT/UTC. Check your system time zone 
   settings if needed.

Q: Cannot edit band or mode
A: This feature was added in v1.2.0. Make sure you're using 
   the latest version.

Q: Retro theme colors don't apply
A: Try switching to another theme and back to retro. The colors 
   should update properly.
